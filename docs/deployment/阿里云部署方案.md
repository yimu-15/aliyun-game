## 一、部署架构

```
                              ┌─────────────────────────┐
                              │      用户 (浏览器)        │
                              └────────────┬────────────┘
                                           │ HTTPS
                                           ▼
                              ┌─────────────────────────┐
                              │   阿里云 CDN / DCDN      │
                              │   (静态资源加速)          │
                              └────────────┬────────────┘
                                           │
                              ┌────────────┴────────────┐
                              │   阿里云 SLB (可选)      │
                              │   负载均衡 + HTTPS 卸载   │
                              └────────────┬────────────┘
                                           │
                    ┌──────────────────────┼──────────────────────┐
                    │                      │                      │
                    ▼                      ▼                      ▼
           ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
           │   ECS / ECI   │      │   ECS / ECI   │      │   ECS / ECI   │
           │  Streamlit    │      │  Streamlit    │      │  FastAPI      │
           │  (前端)       │      │  (冗余)       │      │  (后端API)    │
           │  Port: 8501   │      │  Port: 8501   │      │  Port: 8000   │
           └──────┬───────┘      └──────┬───────┘      └──────┬───────┘
                  │                     │                      │
                  └─────────────────────┼──────────────────────┘
                                        │
                          ┌─────────────┴─────────────┐
                          │       阿里云 OSS           │
                          │    (数据文件 + 日志归档)     │
                          └───────────────────────────┘
                                        │
                          ┌─────────────┴─────────────┐
                          │   阿里云 SLS (日志服务)     │
                          │   (实时日志 + 告警 + 看板)  │
                          └───────────────────────────┘
```

**架构要点:**
- **MVP 部署**: 单台 ECS 同时运行 Streamlit + FastAPI (最低成本)
- **生产部署**: SLB + 多实例 + CDN + 日志/监控
- **数据持久化**: 数据文件存储在云盘或 OSS
- **日志收集**: SLS 统一采集，支持告警

## 三、本地到云端迁移步骤

### Step 1: 项目容器化

```bash
# 在本地 (D:/aliyun-game) 构建镜像
docker build -t worldcup-predictor:latest .
docker tag worldcup-predictor:latest registry.cn-hangzhou.aliyuncs.com/<namespace>/worldcup-predictor:latest
```

### Step 2: 推送到阿里云容器镜像服务 (ACR)

```bash
# 登录 ACR
docker login --username=<阿里云账号> registry.cn-hangzhou.aliyuncs.com

# 推送镜像
docker push registry.cn-hangzhou.aliyuncs.com/<namespace>/worldcup-predictor:latest
```

### Step 3: 创建 ECS 实例

```bash
# 通过阿里云控制台或 CLI 创建
# 推荐: CentOS 7.9 / Alibaba Cloud Linux 3 / Ubuntu 22.04
# 规格: ecs.c7.large (2vCPU 4GB)
# 系统盘: 40GB ESSD
```

### Step 4: ECS 环境初始化

```bash
# SSH 登录 ECS
ssh root@<公网IP>

# 安装 Docker
curl -fsSL https://get.docker.com | bash -s docker
systemctl enable docker && systemctl start docker

# 安装 docker-compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

### Step 5: 拉取并启动

```bash
# 创建项目目录
mkdir -p /opt/worldcup-predictor/data/{raw,processed,logs}
cd /opt/worldcup-predictor

# 拉取 docker-compose.yml
wget https://raw.githubusercontent.com/<repo>/main/docker-compose.yml

# 或直接上传
scp docker-compose.yml root@<IP>:/opt/worldcup-predictor/
scp .env.production root@<IP>:/opt/worldcup-predictor/.env

# 启动
docker-compose up -d
```

### Step 6: 验证

```bash
curl http://localhost:8501/_stcore/health
curl http://localhost:8000/api/health
```

## 五、启动命令

### 方案 A: docker-compose (推荐)

```bash
docker-compose up -d              # 后台启动
docker-compose ps                  # 查看状态
docker-compose logs -f streamlit   # 查看日志
docker-compose down                # 停止
```

### 方案 B: Docker 单独启动

```bash
# Streamlit 前端
docker run -d --name wc-streamlit -p 8501:8501 \
  -v /opt/worldcup-predictor/data:/app/data \
  -e STREAMLIT_SERVER_ADDRESS=0.0.0.0 \
  registry.cn-hangzhou.aliyuncs.com/<ns>/worldcup-predictor:latest \
  streamlit run app/main.py --server.port 8501

# FastAPI 后端
docker run -d --name wc-backend -p 8000:8000 \
  -v /opt/worldcup-predictor/data:/app/data \
  registry.cn-hangzhou.aliyuncs.com/<ns>/worldcup-predictor:latest \
  uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

### 方案 C: 函数计算 FC (Serverless)

```bash
# 使用 s 工具部署
s deploy
```

## 六、域名与访问方式

### 域名配置

```
1. 购买域名 (如 worldcup-predict.cn) — 可选，ECS 公网 IP 即可
2. 在阿里云 DNS 添加 A 记录:
   @    A    <ECS公网IP>
   api  A    <ECS公网IP>
3. 申请免费 SSL 证书 (阿里云 SSL 证书服务)
4. 配置 Nginx 反向代理 + HTTPS
```

### Nginx 配置示例

```nginx
server {
    listen 443 ssl;
    server_name worldcup-predict.cn;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    # Streamlit 前端
    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # FastAPI 后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 访问地址 (答辩展示用)

| 服务 | 地址 |
|------|------|
| 前端页面 | `http://<公网IP>:8501` |
| API 文档 | `http://<公网IP>:8000/docs` |
| 健康检查 | `http://<公网IP>:8000/api/health` |
| (配置域名后) | `https://worldcup-predict.cn` |

## 七、日志与监控建议

### 日志采集

```yaml
# docker-compose.yml 中配置
logging:
  driver: "json-file"
  options:
    max-size: "10m"
    max-file: "3"
```

### 阿里云 SLS 集成 (可选)

```python
# 在 app 中添加 SLS 日志
import logging
from aliyun.log import LogClient

client = LogClient(SLS_ENDPOINT, access_key_id, access_key_secret)
client.put_logs(SLS_PROJECT, logstore, log_items)
```

### 健康检查

```bash
# 添加到 crontab (每 5 分钟检查)
*/5 * * * * curl -f http://localhost:8000/api/health || systemctl restart docker-compose
```

### 云监控

- ECS 基础监控 (CPU/内存/磁盘/网络) — 免费
- 自定义监控指标 (API 响应时间 / 预测请求数)
- 告警规则: CPU > 80% 持续 5 分钟 → 短信通知

## 十、答辩展示要点

### "已部署可访问" 如何展示

**方案 A: 现场打开浏览器 (最佳)**

> 答辩现场直接打开 `http://<公网IP>:8501`，展示 Streamlit 页面。
> 评委可当场操作: 调节参数、查看赛程树、下载报告。

**准备步骤:**
1. 答辩前 1 天: 确保 ECS 实例运行正常
2. 答辩前 30 分钟: 重启容器 `docker-compose restart` 确保缓存新鲜
3. 答辩时: 直接在投影/投屏上展示

**方案 B: 录屏 + 截图 (备选)**

> 如果网络不稳定，录制一段云端操作视频 + 截图作为 materials。

**方案 C: 展示终端 (加分)**

